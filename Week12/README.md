# Полиморфизъм. Статично и динамично свързване. Виртуални функции. Абстрактни класове. Колекция от полиморфни класове

### Статично свързване
* взима се решение за избора на функция по време на компиалция (*compile time*)

```c++
class Base {
    int x;
    void f();
};

class Der : public Base {
    int x;
    void f();
};

int main() {
    Der d;
    d.x // Der::x
    d.f(); // Der::f()

    Base* ptr = &d;
    ptr->x // Base::x
    ptr->f(); // Base::f()
}
```

### Динамично свързване
* взима се решение за избора на фунцкия по време на изпълнение (*runtime*)
* става чрез **виртуални фунцкии**

**virtual** <тип> <име>(<аргументи>) {...}

```c++
class Base {
    virtual void f(); // оказва, че функцията може да бъде презаписана
};

class Der : public Base {
    void f() override; // оказва, че наследника презаписва f()
    // ако сме написали override, програмата търси функция, означена с virtual в базовия клас - ако не я намери хвърля грешка
};

int main() {
    Der d;
    d.f(); // Der::f()

    Base* ptr = &d;
    ptr->f(); // Der::f()
}
```

## Полиморфизъм
Едни и същи действия се изпълняват по различен начин, в зависимост в/у кой обект се прилагат
* действията се наричат **полиморфни**
* реализират се чрез виртуални функции
* класовете, върху които се прилага, трябва да имат общ родител

**¡При полиморфна йерархия ще изтриваме обектите чрез указатели от базовия клас - деструктора задължително е виртуален!**

### Виртуални таблици
Таблици, следящи кой указател/метод да се извика

![](/Week12/resources/virtual_tables.png)

```c++
class A {
    virtual int f();
    virtual int g();
};

class B : public A {
    int f() override;
};

class C : public B {
    int f() override;
    int g() override;
};

int main() {
    A* ptr1 = new B();
    A* ptr2 = new C();

    ptr1->f(); // B::f()
    ptr1->g(); // A::g()

    ptr2->f(); // C::f()
    ptr2->g(); // C::f()
}
```

Това следене на методи се реализира чрез **виртуален ponter** (vPointer)

![](/Week12/resources/polymorphic_f.png)

```c++
int main() {
    A* ptr = new C();
    ptr->f();
}
```

1. първо гледа дали има A::f()
2. ако има virtual, гледа надясно по наследниците
3. ако има друго f(), взима него
4. повтаря се, докато достигнем най-дясната override-ната f()

**¡vPointer-а се запазва най-горе в базовия клас/класа, използвал virtual -> съответно има размер, който влияе на размера на класа!**

### Ключовата дума final
```c++
class Base {
    virtual f();
}

class Der : public Base {
    f() override;
}

class Der2 : public Der {
    f() override;
}

class Der3 : public Der2 {
    f() final; // по-надолу в йерархията няма да можем да презапишем метода; това е финалната версия на f()
}

class Der4 : public Der3 {
    f() override; // няма да се компилира, понеже в Der3 f() е final
}
```

## Абстрактни класове
* клас, който е предназначен за наследяване
* чисто виртуална функция (*pure virtual function*)
    * виртуална функция без тяло
    * предназначена за наследяване
    * няма имплементация в базовия клас; **задължава наследниците да я имплементират**
    * **virtual ... = 0**
* един клас е абстрактен, ако има поне една pure virtual функция
* *ако наследник не override-не чисто виртуална фунцкия на родителя, то и самия наследник става абстрктен клас*

## Колекция от обекти в полиморфна йерархия

Можем да реализираме колекция от различни типове (*но с общ базов клас*) **чрез масив от указатели**

```c++
class Base {
    virtual void f();
}

class A : public Base { 
    void f() override;
}

class B : public Base { 
    void f() override;
    void g();
}

class C : public Base { 
    int t();
}

int main() {
    Base** arr = new Base*[3];
    arr[0] = new A();
    arr[1] = new B();
    arr[2] = new C();

    arr[1]->f() // B::f
}
```

### Възможни проблеми при копиране
Нека имаме клас `Fruit` и негови наследници `Apple`, `Lemon`, `Orange`. Пазим ги в колекция `Fruit** fruits`, която искаме да копираме.

![](/Week12/resources/fruit_example.png)

Проблема, показан на картинката е, че ние имаме колекция `Fruit**` - при самото копиране ние вече не знаем конкретно с какви обекти работим, знаем единствено, че имаме `Fruit*`. Съответно в такъв случай можем да ги създадем/копираме само използвайкки `new Fruit()`. В случая обаче, ако копираме `Apple` (или който и да е клас наследник), ще загубим функционалностите, които класът е надградил над `Fruit`.

Тоест искаме да можем да копираме обектите, без да се интересуваме от конкретния им тип (`Apple`, `Orange`, `Lemon` и т.н.)

*Решението*: в базовия клас въвеждаме метод:
**virtual <базов клас>\* clone() const = 0**

Този метод разписваме във всеки наследник:
```
<базов клас>* clone() const override
{
    return new <наследник>(*this);
}
```

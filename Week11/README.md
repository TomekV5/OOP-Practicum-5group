# Наследяване

Често се случва като пишем даден клас, напр. `B`, да имаме много общи компоненти или поведение с вече съществуващ клас, напр. `A`. Вместо да повтаряме код можем да направим *B наследник на A*

```c++
class A {
    int a;
public:
    void f() {};
};

class B : public A {
    char c; 
    // в момента в B имаме две член-данни -> int a и char c
};

int main() {
    B b;
    b.f();

    return 0;
}
```

![](/Week11/resources/inheritence.png)

* при създаване на обект от тип `B` се създава и обект от тип `A`, който е част от `B`
* наследяват се член-данните и методите на базовия клас
* получава се достъп до някои от наследените компоненти (в зависимост от нивото на достъп)

## Модификатори за достъп и видове наследяване
Възможни модификатори за достъп:
* *private*
    * по подразбиране за класовете
    * недостъпно за външния свят
* *protected*
    * достъп имат единствено класът и наследниците му
* *public*
    * по подразбиране за структурите
    * достъп имат всички класове и обекти

Общ синтаксис при наследяване:

**class <наследник> : <модификатор за достъп> <клас родител>**

Нека имаме
```c++
class A {
private:
    int x;
protected:
    int y;
private:
    int z;
};
```

* *public наследяване*
    * обикновено се използва този тип наследяване
```c++
class B : public A {
    // x -> остава private, като B не може да го достъпи
    // y -> остава protected
    // z -> остава public
};
```

* *protected наследяване*
```c++
class B : protected A {
    // x -> остава private, като B не може да го достъпи
    // y -> остава protected
    // z -> вече е protected
};
```

* *private наследяване*
    * по подразбиране е от този тип
```c++
class B : private A {
    // x -> остава private, като B не може да го достъпи
    // y -> вече е private, достижимо от B, но не и от външния свят
    // z -> вече е private, достижимо от B, но не и от външния свят
};
```

## Подаване на функции
Наследниците може да се подават на функции, приемащи базовия клас, **като можем да използваме само функционалности на базовия клас**

```c++
class A {
public:
    int x;
};

class B : public A {
public:
    int y;
};

void func1(A& obj) {}
void func2(A* ptr) {}
void func3(B& obj) {}

void main()
{
    A a;
    B b;

    func1(a); // няма проблем
    func1(b); // няма проблем

    func2(&a); // няма проблем
    func2(&b); // няма проблем

    func3(b); // няма проблем
    func3(a); // проблем - не можем да подадем родител на клас, очакващ наследник

    return 0;
}

```

![](/Week11/resources/b_visualized.png)

Това, което се получава е, че подаваме `A` частта на `B`. Съответно, ако наследяването е *protectd* или *private*, то `A` частта на `B` е недостъпна за външния свят - в такива случаи не можем да подаваме B на фунцкии, приемащи A

Все пак внимавайте какво подавате на какви функции:
```c++
class A {
private:
    int x;
public:
    A(int x) : x(x) {}

    int& getX() {
        return x;
    }

    const int& getX() const {
        return x;
    }
};

class B : public A {
private:
    int y;
public:
    B() : B(0, 0) {}
    B(int x, int y) : A(x), y(y) {}
};

void g(A* ptr) {
    ptr[0].getX()++;
    ptr[1].getX()++;
}

int main() {
    B* b = new B[3];
    g(b);

    for (size_t i = 0; i < 3; i++) {
        std::cout << b[i].getX() << std::endl;
    }

    delete[] b;
    return 0;
}

```

Очакваме резултат '1 1 0', но всъщност е '1 0 0'. Защо? Нека видим как масивът се пази в паметта:

![](/Week11/resources/triple_B.png)

Имаме последователност от A и y, които в конкретния случай имат равен размер, равен на размера на един `int`. Както се вижда, `g` приема `A* ptr`, така че `prt[0]` и `prt[1]` всъщност са отмествания от началото на масива **с дължината на A (sizeof(int))**. Така ние все още не сме достигнали до второто B и то остава непроменено

## Конструктори и деструктори
* *конструктори*
```c++
class A {
    int x;
    // ...някаква реализация...
};

class B : public A {
    int y;

    B(int y) {
        this->y = y;
        // нямаме нищо оказано за x, така че търсим default-ен конструктор на А
        // в случая, в който нямаме конструктор по подразбиране за A (разписали сме друг или сме му дали delete), тук програмата ще даде грешка
    }

    B(int x, int y) : A(x) // така може да окажем кой точно контруктор на A да се извика (стига да съществува разбира се)
};
```

* *деструктори* - деструктора на B вика автомаично деструктора на A

## Копиране при наследяване
```c++
B::B(const B& other) : A(other) // копиращия конструктор е вид конструктор, така че просто го викаме в инициализиращия списък 
{
   copyFrom(other);
}

B& B::operator=(const B& other)
{

   if (this != &other)
   {
   	A::operator=(other); // operator= вече не е конструктор, така че не можем да го извикаме в initializer list-a; трябва да го извикаме експлицитно в кода по този начин
   	free();
   	copyFrom(other);
   }
   return *this;
}
```

## Дублиране на функции и член-данни
```c++
class A {
    void f() {...}
};

class B : public A {
    void f() {...}
}

B obj // -> сочи цялото B, съотв. и неговото f() (виж картинката)
A& ref = obj // -> това вече сочи само към A частта на B, съответно и към A::f() 
```

```c++
class A {
private: 
    int a;
public:
    void change_a() { a++; }
};

class B : public A {
private: 
    int a;
public:
    void print_a() { std::cout << a; }
}

int main() {
    A a;
    B b;

    return 0;
}
```

`change_a()` е в А и вижда само `A::a`, съответно променя него. Въпреки че `b.change_a()` е напълно валидна операция, тя ще промени `A::a`, `B::a` остава непроменено.

Съответно `b.print_a()` принтира `B::a`, което няма да е отразило промените на `b.change_a()`
